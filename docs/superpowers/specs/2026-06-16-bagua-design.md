# 八卦占卜 + 八字算命 设计文档

- 日期：2026-06-16
- 状态：第一期（MVP）已实现并验证

## 1. 目标

提供"金钱卦起卦"与"八字排盘"两大功能的 Web 应用，用于传统文化研究。
第一期不依赖 AI，仅凭命理算法 + 静态卦辞/八字数据即可闭环。

## 2. 范围与非目标

- **范围内（第一期）**：金钱卦起卦（后端掷币、朱熹动爻解卦、64 卦卦辞/彖象/爻辞）、八字四柱排盘（公历/农历输入、五行统计、纳音、大运起运信息）。
- **非目标（第二期）**：AI 个性化解读、用户账号/历史记录、流年逐年排布、生产部署。

## 3. 架构

分层单仓：

```
backend/  Python FastAPI，核心算法为纯函数（app/core/）
frontend/ React + Vite + TypeScript，经 vite proxy 调用后端
```

核心原则：
- `app/core/` 是纯函数，无 IO、无框架依赖，可独立单测、可命令行验证。
- `app/api/` 是薄路由层，只做 HTTP↔core 转换。
- `app/core/data/64hexagrams.json` 为数据驱动，由 `scripts/build_hexagrams.py` 生成并自检。

## 4. 金钱卦算法

### 4.1 起卦

三枚铜钱摇一次，数背面数：
- 0 背 → 老阴(6，动，阴)
- 1 背 → 少阳(7，静，阳)
- 2 背 → 少阴(8，静，阴)
- 3 背 → 老阳(9，动，阳)

摇六次得六爻。随机源用 `secrets.SystemRandom`（密码学级，不可预测），起卦由后端生成，前端不传硬币结果，保证公正性。

### 4.2 本卦与变卦

6 爻阴阳 → 查 64hexagrams.json 得本卦；动爻翻转 → 变卦。

**卦序采用文王卦序**（乾=1, 坤=2 ...），与"6 爻当二进制"的二进制序不同，因此查卦走 lines 模式查表，不做二进制计算。

### 4.3 64 卦数据生成

`build_hexagrams.py` 用语义化卦名（"震""坎"…）查 `TRIGRAM_LINES` 表组合成 6 爻，彻底脱离二进制位序歧义。生成时做唯一性自检（64 卦 lines 两两不同）。

> 历史教训：早期版本用 `(lower, upper)` 二进制数值推导 lines，因位序约定不一致导致 10 组卦冲突。改用语义化查表后消除。

### 4.4 解卦（朱熹《易学启蒙》动爻法）

| 动爻数 | 规则 |
|--------|------|
| 0 | 以本卦卦辞断之 |
| 1 | 以本卦动爻爻辞断之 |
| 2 | 以本卦两动爻爻辞断之，以上爻为主 |
| 3 | 以本卦卦辞 + 变卦卦辞合断 |
| 4 | 以变卦两不动爻爻辞断之，以下爻为主 |
| 5 | 以变卦不动爻爻辞断之 |
| 6 | 乾用九 / 坤用六；其余以变卦卦辞断之 |

## 5. 八字算法

基于 `lunar-python` 的 `Solar`/`Lunar` 与 `EightChar`。完整排盘覆盖业界"基础排盘必备"档：

- **四柱**：年/月/日/时柱天干地支（公历/农历输入）
- **日主**：日柱天干（论命之主）
- **天干/地支五行阴阳**
- **地支藏干**（人元：主气/中气/余气）—— `EightChar.getDayHideGan()` 等
- **十神**：天干十神 + 藏干十神（以日主为基准的生克关系）—— `getDayShiShenGan()` / `getDayShiShenZhi()`
- **纳音**：每柱纳音（如"海中金"）—— `getDayNaYin()` 等；另提供按"干支串"反查的六十甲子纳音表
- **空亡**（旬空）与 **旬** —— `getDayXunKong()` / `getDayXun()`
- **十二长生**（长生/沐浴/.../养）—— `getDayDiShi()` 等
- **三垣**：胎元、命宫、身宫 —— `getTaiYuan()` / `getMingGong()` / `getShenGong()`
- **五行统计**：天干 + 地支本气，共 8 位
- **大运**：起运岁数 + 顺逆 + 每柱干支 + 对应年龄区间 + 起始年份 —— `getYun().getDaYun()`
- **流年**：指定年份干支 + 流年天干十神（相对日主）
- **六亲对应关系**：年柱祖上父母、月柱父母兄弟、日柱自身配偶、时柱子女
- **常用神煞**（自写查表，非黄历神煞）：天乙贵人、文昌、禄神、羊刃（以日干查）；驿马、桃花、华盖（以年支/日支查，三合局规则）

> 注：日主旺衰综合判断、格局判定、用神/喜忌神推算属"进阶解析"，业界无算法共识，
> 留待第二期交给 LLM，不硬编码进排盘核心。

## 6. API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/divine/cast` | 起卦（后端掷币） |
| GET  | `/api/divine/hexagram/{num}` | 查单卦详情（1-64） |
| POST | `/api/bazi/chart` | 八字排盘 |
| GET  | `/api/health` | 健康检查 |

统一错误格式：`{"detail": {"code": "...", "message": "..."}}`。

## 7. 测试策略

- `core/` 100% 单测覆盖，含历史回归用例：
  - 金钱卦：经典卦例（乾之姤、乾变坤、用九用六、既济/未济卦序）
  - 八字：毛泽东 1893-12-26 辰时（癸巳/甲子/丁酉/甲辰）
- `api/` 用 FastAPI TestClient 集成测试。

## 8. 关键设计决策

1. 起卦由后端生成（SystemRandom），防作弊且公正。
2. 第一期卦辞/爻辞用静态 JSON 数据，不依赖 AI 即可闭环。
3. core/ 纯函数无 IO，可命令行验证、单测无需 mock。
4. 64 卦 lines 由语义化卦名查表，避免二进制位序歧义。
5. 卦序用文王卦序（符合用户认知"乾为第一卦"），查卦走 lines 查表。
6. LLM 留待第二期，做成可插拔 Protocol 抽象。

## 9. 验证结果（第一期 + 八字扩展）

- 后端：50 个测试全绿（iching 14 + bazi 26 + API 11）。bazi 测试含完整排盘字段回归（藏干、十神、空亡、十二长生、三垣、大运区间连续性、流年十神、神煞）。
- 端到端：扩展后 `/api/bazi/chart` 返回完整排盘，1990-01-01 午时验证（丙日主，藏干甲丙戊，藏干十神偏印比肩食神，十二长生长生，三垣胎元丁卯/命宫乙亥/身宫辛未，大运首柱乙亥 9-18 岁，流年 2024 甲辰偏印/2025 乙巳正印，神煞禄神+羊刃+桃花）。
- 前端：`npm run build` 类型检查通过；八字页新增排盘详情表、三垣、大运时间轴、流年、神煞、五行分布等板块。
