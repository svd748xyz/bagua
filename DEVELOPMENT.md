# 玄机阁 · 项目开发文档

> 八卦占卜 + 八字排盘 Web 应用
> 最近更新：2026-06-17

---

## 一、项目背景

中国传统命理学（周易占卜、八字四柱）有完整的理论体系，但现有在线排盘工具质量参差：
- 多数要么只有排盘数据、缺少解读，要么解读全靠人工、无法规模化
- 开源命理库（如 lunar-python、sxtwl）算法扎实，但缺一个把它们串成好用的 Web 产品的工程层
- 排盘数据与 AI 解读通常耦合在一起，难以各自演进

本项目旨在用现代 Web 技术栈，把**命理算法（确定性、可验证）**和**AI 解读（生成性、可演进）**解耦，做一个既专业准确、又能持续增强解读能力的产品。

核心原则：**排盘引擎只负责"事实"（算准四柱、起准卦象），解读交给上层**。这样命理算法可被严格回归测试，解读能力可独立迭代。

## 二、最终目标

一个面向中文用户的命理 Web 应用，分两期：

**第一期（已基本完成）—— 确定性排盘**
- 金钱卦起卦：模拟三枚铜钱摇六次，得本卦/变卦，按朱熹《易学启蒙》动爻法解卦，含完整 64 卦卦辞/彖传/大象传/六爻辞
- 八字排盘：输入公历/农历生辰，排出**业界"基础排盘必备"档**的完整信息（四柱、藏干、十神、纳音、空亡、十二长生、三垣、大运、流年、神煞、五行分布）
- 不依赖任何 AI，仅凭算法 + 静态数据即可完整闭环

**第二期（规划中）—— AI 增强解读**
- 可插拔 LLM 层（Protocol 抽象），接入卦象/八字的个性化解读
- 流式输出（SSE），带命理免责声明
- 进阶命理分析（刑冲合害、五行强弱、格局、用神喜忌）——这些业界无算法共识，适合交给 LLM

**明确不做（至少当前阶段）**：用户账号/历史记录、商业化、生产级部署运维、移动端 App。

## 四、技术架构

### 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 命理核心 | Python 纯函数 | `app/core/`，无 IO/无框架依赖，可独立单测、可命令行验证 |
| 后端 | FastAPI + Pydantic + lunar-python | 薄路由层，只做 HTTP↔core 转换 |
| 前端 | React 18 + Vite + TypeScript + Framer Motion | 经 vite proxy 调后端 |
| 架构 | 分层单仓（backend/ + frontend/） | 核心算法与展示解耦 |

### 核心设计决策

1. **核心算法纯函数化**：`app/core/` 不依赖 FastAPI/网络/数据库，可命令行直接 import 验证，单测无需 mock。这是项目最值钱的部分，换掉 Web 框架也不受影响。
2. **起卦由后端生成**：用 `secrets.SystemRandom`（密码学级随机）掷币，前端不传硬币结果，保证公正性、防作弊。
3. **64 卦数据驱动**：lines 由语义化卦名（"震""坎"…）查 `TRIGRAM_LINES` 表组合，生成时做唯一性自检。避免二进制位序歧义（曾因位序约定不一致导致 10 组卦冲突，已修复）。
4. **卦序用文王卦序**：符合用户认知"乾为第一卦"，查卦走 lines 模式查表，不做二进制计算。
5. **八字神煞自写查表**：lunar-python 的神煞是黄历择日用，与命盘神煞不同；按日干/年日支查表实现天乙/文昌/禄神/羊刃/驿马/桃花/华盖。
6. **进阶命理留给 LLM**：日主旺衰、格局、用神业界无算法共识，不硬编码进排盘核心，第二期交给 LLM。
7. **启动逻辑用 Python**：Windows bat 的 `start`/编码/换行坑多，复杂启动逻辑放 `launcher.py`，bat 只做单行委托。

## 八、关键历史教训

1. **64 卦位序冲突**：早期用 `(lower, upper)` 二进制数值推导 lines，因位序约定（bit0 vs bit2 = 初爻）不一致，导致 10 组卦 lines 冲突、查表丢失。改用语义化卦名查表后彻底解决。教训：涉及"二进制 ↔ 传统表示"映射时，直接用查表，不要推导。

2. **bat 文件编码**：中文 Windows cmd 按 GBK 解读 bat 文件，UTF-8 编码的 bat 会乱码成 `此时不应有 ...` 导致脚本崩溃。所有 bat 必须用 GBK + CR（`\r`）换行。

3. **`start` 命令派生窗口**：cmd 的 `start /min` 在某些 Windows 会话（非交互、特定安全策略）下会派生子窗口失败（报"另一个程序正在使用此文件"）。复杂启动逻辑应放 Python（`subprocess.Popen`），bat 只做单行委托。
